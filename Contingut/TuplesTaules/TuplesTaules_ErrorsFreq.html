7.6 ERRORS I PROBLEMES FREQÜENTS

Accions i funcions amb taules:

En C++, el nom d'una taula és directament una referència, així que les taules sempre es passen per referència. Per motius de coherència, no s'ha de posar mai el símbol & en els paràmetres formals corresponents.. Així, una taula no la podem passar per valor, però, si s'escau, podem fer que sigui considerada com un paràmetre d'entrada constant indicant al compilador que no poden ser modificats els valors originals; això es fa afegint const davant dels paràmetres formals, com per exemple: 
void mostrar_vector(const Vector_enter vector, unsigned n)
Podem passar una taula a una funció, però no podem retornar una taula!; en aquest cas, podem utilitzar una acció.

Assignació de taules

Una variable de tipus taula guarda l'adreça del primer element de la taula, per això, per copiar-les, no ho podem fer amb l'operador d'assignació. Dóna error, quan és una taula d'elements bàsics, no còpia elment a element, el compilador es queixa. Per fer la còpia podem fer una acció que se'n encarregui: 
void copia(const Vector_enter t1, Vector_enter t2, unsigned n){
		//Pre: 0<=n<=N_MAX
		//Post: t2[0..n] = t1[0..n]
			for(unsigned i=0; i<n; i++) t2[i]=t1[i];
		}


Pas de vectors i matrius

En lloc d'usar una definició de tipus vector/matriu a la capçalera de l'acció o de la funció, podríem fer-ho de la següent manera: 
//Per vectors:
		void llegir_vector_enter(int vec[], unsigned n) {
    //Pre: 0<=n<=N_MAX
    //Post: vec_enter[0..n-1] conté successivament els enters entrats
    for (unsigned i=0; i<n; i++) cin>>vec[i];
}

int suma_vector_enter(const int vec[], unsigned n) {
    //Pre: 0<=n<=N_MAX
    //Post: retorna suma de vec[0..n-1]
    int suma=0;
    for (unsigned i=0; i<n; i++) suma+=vec[i];
    return suma;
}
