5.5 ESTIL I BONES PRÀCTIQUES

Implementació de subprogrames

Per implementar un subprograma cal:
  • Escollir un nom representatiu pel subprograma.
  • Determinar si és convenient que sigui una acció o una funció.
  • Si és una funció, identificar el tipus del valor retornat.
  • Determinar els paràmetres del subprograma i donar-los un nom significatiu i el tipus adequat.
  • Fer l'especificació.
En els fitxers .cpp, la implementació dels subprogrames es col·loca abans de l’int main(); la implementació d’un subprograma ha d’aparèixer abans de que sigui cridat.

Bones pràctiques

Validar la correctesa dels subprogrames fent un int main() exclusivament destinat a validar el funcionament del subprograma.
Documentar bé els subprogrames donant una explicació del significat dels paràmetres (si no és evident) i l'especificació que no ha de duplicar la informació de la capçalera, i ha d'explicar el què i no el com.
Dissenyar una estratègia per resoldre el problema, buscant sempre crear subprogrames curts (en general, que no ocupin més d'una pantalla) i intentar fer-los el més genèrics i reutilitzables possible.

Noms i identació

Els noms dels subprogrames comencen en minúscules i s'escriuen utilitzant l'estil camel o snake. Han de ser descriptius (indicar què fa el subprograma).
Els noms dels paràmetres han de3 ser significatius (en el si del subprograma) i els algorismes s'han d'escriure amb claredat.
El codi s'ha d'identar(La capçalera, l'especifcació i la clau que tanca el subprograma estan alienats, i el codi identat cap a la dreta). 
bool es_primer(unsigned nombre) {
//Pre: --
//Post: cert si nombre és primer
    unsigned aux=2;
    bool es_primer=nombre>1; //es considera primer d’entrada, tot nombre >1
    while(es_primer and aux*aux<=nombre) {
        es_primer=nombre%aux!=0;
        aux++;
    }
    return es_primer;
}

Entrons globals

Constants Globals

El C++ ens permet definir constants globals que són visibles des de tot arreu. Són recomanables per evitar duplicacions. Es defineixen al principi del fitxer sense estar a dins de cap bloc de codi. Quan una constant es necessita en diferents parts del codi, es recomana utilitzar constants globals. 
//Directives del compilador
#include<iostream>
using namespace std;

//Definició de constants globals
const double PI=3.14159;

//Implementació de subprogrames
double volum_esfera(double costat) { ... }

double superficie_esfera(double radi) { ... }

//Implementació del programa principal
int main() {...}

Ús de variables globals enlloc de pas de paràmetres

El C++ permet declarar variables globals; una vegada declarades (a fora de tots els blocs de codi) són accessibles des de tot arreu. Les variables globals poden ser modificades des de qualsevol lloc i no es té control d’on es modifiquen. Els subprogrames tenen accés a les variables globals i per tant, no és tant fàcil saber què fan (quines variables utilitzen o modifiquen), deixant de ser caixes negres que treballen amb el paràmetres que reben i complicant la comprensió del codi. Cal evitar l’ús de variables globals!

Prototipus

El C++ permet usar prototipus; és a dir, es poden escriure primer les capçaleres dels subprogrames i implementar-los més tard. S'escriuen les capçaleres seguides de ; al principi del fitxer.
En els prototipus els noms dels paràmetres són opcionals, encara que és recomanable posar-los per tal que ajudin a identificar els paràmetres.
Usar prototipus implica duplicar les capçaleres (apareixen en el prototipus i en la implementació); si hi ha canvis, cal fer-los per duplicat 
double volume_sphere(double radius); //prototipus
void some_geometry(); //prototipus
...
void some_geometry() {//implementació
    double v=volume_sphere(1.0);
    ...
}

double volume_sphere(double radius) {//implementació
    return 4.0*Pi*radius*radius*radius/3.0;
}
