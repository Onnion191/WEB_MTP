ERRORS I PROBLEMES FREQÜENTS

Tipus de sortida dels procediments

El return d'una funció o la "sortida" del pas per referència i la sortida per pantalla són coses diferents:
Sortida per pantalla <<
  • Es mostra qualcom a l'usuari des del (sub)programa.
  • Permet la comunicació amb l'usuari però no la comunicació interna en el programa

Pas per referència no constant o return
  • Permet la comunicació interna en el programa
  • el resultat de les instruccions executades en el subprograma s'utilitza en el (sub)programa on s'ha fet la crida

Funció sense return

Tota funció ha de retornar sempre un valor (del tipus indicat). És un error comú olbidar-se'n. En aquest cas pot aparèxer un warning al compilar, cosa que no sempre passa; i la funció retorna un valor aleatori.
La derrera instrucció duna funció és convenient que sigui un return: 
double radi (double area_cercle){
	if (area_cercle>=0){
		return sqrt(area_cercle/PI);
	}//Que passa si el valor de area_cercle és negatiu?
}
Es recomana, en general, que hi hagi un sol return per funció.
