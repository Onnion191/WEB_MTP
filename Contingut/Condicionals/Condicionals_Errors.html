3.5 ERRORS FREQÜENTS

Errors i problemes freqüents

A continuació farem un llistat dels errors més freqüents en programació de condicions i estructures alternatives:

Mal ús dels punt i coma

Hem vist que en C++ és necessari acabar les instruccions amb ;. Però això no s'aplica amb les estructures, aleshores el següent codi és un error: 

if(condicio);
	fer_quelcom
En aquest cas el ;, està mal col·locat, però no és un error de programació ja que és un sentència vàlida. En C++, utilitzar el ;, significa que no es vol fer res. Per tant en aquest cas estem dient que si es compleix la condició del if no volem fer res. I a continuació de "no fer res", s'executarà el fer quelcom.

Ara bé, per altra banda, es pot utilitzar el punt i coma de forma intencionada per remarcar que no vols fer res si es dóna el cas. Això significa que com a programador has pensat amb aquella possibilitat, però has determinat que no calia fer cap instrucció.
Un exemple: 

//Entrada: L'usuari entra un valor enter
//Sortida: retorna el valor absolut del nombre.
int digit; 
cout<<"Entra un enter";
cin>>digit;
if(ditigit<0)
	digit*=-1;
else ;//El digit ja es positiu, no cal fer res

Mala indentació

Sabem que la indentació és molt important per saber dins de quina estructura ens trobem. Si no la respectem aleshores ens poden succeir errors dins del nostre codi. Exemple: 

double cost_enviament ; //El cost d'enviament estàndard es de 10€
if(estat == "USA")//Si el paquet prové d'estats units
	if(estat== "CA")//Estat == california
		cost_enviament = 12.0;
else// Aquest else què vol dir? que no sigui California o que no sigui d'Estats Units?
	cost_enviament = 15.0€

Confusió entre igualtat i assignació

Com ja sabem de les variables podem tenir diferents operadors, entre ells hi ha l'Assignació, = i la comparació ==. Els quals tenen propòsits ben diferents, entre ells, un serveix per donar un valor a una variable, mentre que l'altre serveix per comparar dos valors.
Si no els usem correctament, poden passar coses com les següents: 

int a = 5; 
if(a = 2)//En aquest cas no estem comparant res, sinó assignant un valor a A.
{
	cout<<"a val dos"<<endl;
}
else 
{
cout<<"a no val dos"<<endl;
}
Quin seria el resultat de l'execució?

Errors en operadors lògics

També sabem que les variables lògiques permeten fer combinació d'operadors lògics, ara bé, hem d'anar amb compte en no escriure errors ja que després tindrem resultats no desitjats.
Els errors més freqüents que hi ha són els següents: 

int a = 4;
if (a == 3 || 6 ) //Error, la segona condició està avaluant 6 per separat, i per tant sempre donarà true. (6 != 0)
	a = 3; 

//La forma correcte: 
if (a == 3 || a == 6)

int x = 3,y=8;
if(x && y > 0)//Error, x s'avalua sola, i per tant sempre donarà certa la seva part. 
	x = y; 
	
//La forma correcte: 
if ( x > 0 && y>0)
